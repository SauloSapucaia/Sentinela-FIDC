Padrões visuais extraídos do motor Python antes da aplicação de Machine Learning.

Qualidade e Gap de Dados

Identificamos que 46.6% dos dados de liquidez (cedente) estavam nulos. Em vez de descartar os dados e enviesar o modelo, aplicamos engenharia de features com imputação estatística pela mediana.

Duplicatas Transacionais

Alerta Crítico: 14.6% da base corresponde a grupos de boletos com o mesmo pagador, beneficiário, valor e data de emissão, disfarçados com IDs diferentes. Risco imediato de dupla cobrança.

Smurfing e Overpayments

Detectamos 509 boletos pagos com valor muito superior ao nominal. Esse padrão sistêmico é um vetor clássico de lavagem de dinheiro (pagar a mais para pedir devolução limpa em outra conta).

Anomalia Temporal

Houve uma "explosão repentina" de emissões onde 73% dos boletos foram gerados em apenas 2 meses. Isso caracteriza o lançamento retroativo de títulos frios na base.

Perfil Financeiro do FIDC

A mediana de atraso histórico das empresas da base é de impressionantes 178 dias. Isso prova que a carteira operada já possui um nível de "crédito podre" (toxicidade) acima do limite de segurança aceitável.

Concentração e Colusão

Aproximadamente 80% de todo o volume está concentrado em um grupo minúsculo de atores. Existem dezenas de CNPJs que só realizam transações financeiras milionárias entre si mesmos (auto-faturamento).

// 1. ENSEMBLE DE MACHINE LEARNING (IF + LOF)

# Feature Engineering: Winsorização e Escalonamento
scaler = RobustScaler()
X_scaled = scaler.fit_transform(X_features)

# Redução de Dimensionalidade (PCA) preservando a variância
pca = PCA(n_components=15, random_state=42)
X_pca = pca.fit_transform(X_scaled)

# Modelo 1: Isolation Forest (Busca anomalias globais em florestas)
modelo_if = IsolationForest(n_estimators=300, contamination=0.02, random_state=42)
modelo_if.fit(X_pca)

# Modelo 2: Local Outlier Factor (Busca anomalias por densidade local)
modelo_lof = LocalOutlierFactor(n_neighbors=20, contamination=0.02)
lof_scores = modelo_lof.fit_predict(X_pca)

# ENSEMBLE: A fraude é confirmada se capturada por qualquer um dos vetores matemáticos
df['flag_fraude_ia'] = np.where((modelo_if.predict(X_pca) == -1) | (lof_scores == -1), 1, 0)

// 2. EXPLAINABLE AI (TRADUZINDO A MATEMÁTICA PARA O NEGÓCIO)

# Função de negócio para explicar ao analista O PORQUÊ do bloqueio da IA
def _explicar(linha):
    motivos = []
    # Compara o boleto suspeito com o 99º e 95º percentil de toda a base
    if linha['vlr_nominal'] >= refs['vlr_nominal']['p99']:
        motivos.append("Valor do boleto extremamente alto")
    if linha['desvio_ticket_pagador'] >= refs['desvio_ticket_pagador']['p95']:
        motivos.append("Foge absurdamente do padrão de gastos do pagador")
    if linha['freq_dupla_pag_benef'] >= refs['freq_dupla_pag_benef']['p99']:
        motivos.append("Repetição anormal entre estas duas empresas")
        
    return " | ".join(motivos) if motivos else "Combinação multivariada anômala"

# Aplica a justificativa apenas nos casos pegos pelo Ensemble
df['motivo_anomalia_ia'] = np.where(df['flag_fraude_ia'] == 1, df.apply(_explicar, axis=1), 'N/A')

// 3. CONTEXTO MACROECONÔMICO (API) E Z-SCORE SETORIAL

# Conexão Open Data com BACEN para calibrar a tolerância de risco do FIDC
selic_atual = float(requests.get("https://api.bcb.gov.br/dados/...").json()[0]['valor'])
if selic_atual > 10.0:
    df['pontuacao_risco'] += 5 # O crédito está caro, a malha fina aperta.

# Z-Score: Estatística de desvio padrão baseada na média daquele setor específico (IBGE)
df['z_score_setorial'] = df.groupby('cd_cnae_prin_beneficiario')['vlr_nominal'].transform(
    lambda x: (x - x.mean()) / x.std()
)
